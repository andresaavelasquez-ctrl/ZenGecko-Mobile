# v0.1.20 — SearchInput Type Hotfix

GitHub Actions superó recursos, manifiesto y configuración de firma, pero
`compileDebugJavaWithJavac` se detuvo porque el campo estaba declarado como:

```java
private EditText searchInput;
```

La implementación v0.1.20 llama:

```java
searchInput.setSelectionListener(...);
```

Ese método pertenece a `ZenAddressEditText`, no a `EditText`. El hotfix
corrige la declaración a:

```java
private ZenAddressEditText searchInput;
```

No se cambia la versión porque la APK v0.1.20 no fue generada.
